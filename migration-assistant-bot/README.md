# Migration Assistant Bot

## Описание
Чат-бот для автоматизации взаимодействия с клиентами.

---

## Моя роль

- анализ и декомпозиция требований
- проектирование бизнес-процессов (BPMN)
- проектирование интеграций
- описание API
- моделирование данных (ERD)

---

## Ключевые решения

- разделение сценариев по бизнес-веткам (консультация / продажа / заявка)
- использование webhook для обработки оплаты
- единая модель пользователей для всех каналов
- фиксация событий для аналитики

---

## BPMN (бизнес-процесс)
Диаграмма отражает пользовательские сценарии взаимодействия с чат-ботом:  
- онлайн-консультация  
- покупка шаблонов  
- заказ услуги  
- оставление заявки  

![BPMN](./bpmn.png)

Диаграмма выполнена в нотации BPMN 2.0 с разделением по ролям (User / System / Manager).

---

## Sequence Diagram (оплата)

Диаграмма показывает технический процесс оплаты через YooKassa:  
- создание платежа  
- редирект пользователя  
- обработка webhook  
- проверка статуса оплаты  
- дальнейшая бизнес-логика  

![Sequence](./sequence.png)

---

## Основные сценарии

### Онлайн-консультация
Выбор темы → ответ → выбор дальнейшего действия

### Покупка шаблона
Выбор шаблона → оплата → получение PDF → создание лида

### Заказ услуги
Заполнение данных → оплата → создание лида → уведомление менеджера

### Оставление заявки
Анкетирование → оценка → создание лида → работа менеджера

## ERD (модель данных)

Диаграмма отражает ключевые сущности системы: пользователей, платежи, лиды, продукты, запросы на документы, покупки и события аналитики.

![ERD](./erd.png)

---

## Архитектура решения

Система состоит из:

- Web Chat (интерфейс пользователя)
- Backend (бизнес-логика)
- YooKassa (платежи)
- CRM (Bitrix24)
- Telegram (уведомления менеджеру)

Основные потоки:
- пользователь → чат → backend
- backend → платежная система
- backend → CRM
- backend → Telegram
