# API и интеграции — Migration Assistant Bot

## Назначение

В проекте используются интеграции для:

- создания и обработки платежей
- создания лидов в CRM
- уведомления менеджеров
- фиксации результатов оплаты

---

# 1. Платежи

## POST /api/payments

Создание платежа для выбранного продукта или услуги.

### Request

```json
{
  "user_id": "12345",
  "product_type": "template",
  "product_name": "Шаблон заявления на ВНЖ",
  "amount": 1000,
  "currency": "RUB",
  "customer_name": "Иван Иванов",
  "phone": "+79991234567"
}
````

### Response

```json
{
  "payment_id": "pay_001",
  "status": "pending",
  "confirmation_url": "https://yookassa.ru/checkout/..."
}
```

### Бизнес-логика

* endpoint вызывается после выбора продукта и ввода контактных данных
* система создаёт платёж через YooKassa
* пользователю возвращается ссылка на оплату

### Возможные ошибки

* некорректная сумма
* не передан тип продукта
* ошибка создания платежа во внешней системе

### Пример ошибки

```json
{
  "error": "PAYMENT_CREATE_FAILED",
  "message": "Не удалось создать платеж"
}
```

---

# 2. Проверка статуса платежа

## GET /api/payments/{payment_id}

Получение текущего статуса платежа.

### Response

```json
{
  "payment_id": "pay_001",
  "status": "succeeded",
  "amount": 1000,
  "currency": "RUB"
}
```

### Возможные статусы

* pending
* succeeded
* canceled
* failed

---

# 3. Webhook от YooKassa

## POST /api/webhooks/yookassa

Endpoint для получения уведомления от YooKassa об изменении статуса платежа.

### Request

```json
{
  "event": "payment.succeeded",
  "object": {
    "id": "pay_001",
    "status": "succeeded",
    "amount": {
      "value": "1000.00",
      "currency": "RUB"
    }
  }
}
```

### Бизнес-логика

После получения webhook система:

1. проверяет, был ли уже обработан этот `payment_id`
2. запрашивает актуальный статус платежа
3. определяет тип продукта
4. запускает дальнейший сценарий:

   * отправка PDF
   * создание лида
   * уведомление менеджера

### Response

```json
{
  "result": "ok"
}
```

# 4. Создание лида в CRM

## POST /api/crm/leads

Создание лида в CRM после успешной оплаты или после заполнения заявки.

### Request

```json
{
  "user_id": "12345",
  "name": "Иван Иванов",
  "phone": "+79991234567",
  "source": "chatbot",
  "scenario": "template_purchase",
  "product_name": "Шаблон заявления на ВНЖ",
  "payment_id": "pay_001"
}
```

### Response

```json
{
  "lead_id": "crm_98765",
  "status": "created"
}
```

### Бизнес-логика

Лид создаётся в случаях:

* успешная покупка шаблона
* успешная оплата услуги
* успешное прохождение сценария оставления заявки

### Возможные ошибки

```json
{
  "error": "CRM_CREATE_FAILED",
  "message": "Не удалось создать лид в CRM"
}
```

---

# 5. Уведомление менеджера

## POST /api/notifications/telegram

Отправка уведомления менеджеру о новом лиде или покупке.

### Request

```json
{
  "lead_id": "crm_98765",
  "scenario": "service_order",
  "name": "Иван Иванов",
  "phone": "+79991234567",
  "message": "Новый лид после оплаты услуги"
}
```

### Response

```json
{
  "status": "sent"
}
```

---

# Основные бизнес-правила

1. Платёж должен быть успешным до выдачи PDF.
2. Лид в CRM создаётся только после подтверждённой оплаты или после завершения сценария заявки.
3. Один и тот же webhook не должен обрабатываться повторно.
4. После создания лида менеджер получает уведомление в Telegram.
5. Для покупки шаблона пользователь получает PDF автоматически.
6. Для услуги после оплаты инициируется дальнейшая работа менеджера.
